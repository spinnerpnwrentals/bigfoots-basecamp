# Project Overview

Bigfoot's Basecamp is a premium family vacation rental located in Snoqualmie Pass, Washington.

The website is not simply an information site.

It is the digital companion to the guest experience.

Guests should feel like they are opening a modern National Park field journal rather than a generic vacation rental website.

The scavenger hunt is one feature of a much larger guest portal.

The long-term goal is to create one of the most memorable vacation rental websites anywhere.

---

# Brand Personality

The experience should feel:

- Premium
- Warm
- Adventurous
- Authentic
- Relaxing
- Family friendly
- Modern
- Pacific Northwest

Avoid:

- Cartoonish
- Halloween themed
- Cheap camping aesthetic
- Excessive "Bigfoot jokes"
- Bright saturated colors
- Gimmicky design

Bigfoot should be mysterious, playful, and tasteful.

# Technical Requirements

Remain a static website.

No framework unless specifically requested.

Fast loading.

Mobile-first.

Accessible.

Maintain existing scavenger hunt functionality.

Never break:

- localStorage
- progress tracking
- hints
- admin mode
- confetti
- secret guide

---

# Coding Standards

Prefer reusable code.

Avoid duplication.

Keep HTML semantic.

Keep CSS organized.

Keep JavaScript modular.

Do not introduce unnecessary dependencies.

Use lowercase filenames.

Use hyphenated file names.

Never use spaces in filenames.

---

# Git Workflow

Make small logical commits.

Examples:

- Add branding assets
- Improve homepage navigation
- Create guest guide page
- Add restaurant recommendations

Do not create large unrelated commits.

Always summarize planned changes before editing.

Always show the diff before committing.

Never commit unless explicitly approved.

# Decision Principles

When making implementation decisions without explicit user direction:

- Preserve existing functionality unless asked to change it.
- Mobile-first design always takes priority.
- Prefer simplicity over cleverness.
- Reuse existing components whenever practical.
- Maintain a consistent visual language across every page.
- Optimize for fast page loads and minimal dependencies.
- Favor readability and maintainability over micro-optimizations.
- Keep navigation intuitive and consistent.
- Every page should provide a clear path back to the Guest Portal.
- Avoid unnecessary JavaScript when HTML or CSS provides an equivalent solution.
- Keep animations subtle and optional, respecting reduced-motion preferences.
- Preserve accessibility best practices.
- Never break existing scavenger hunt functionality.
- Never remove user data stored in localStorage without explicit instruction.
- When multiple reasonable design choices exist, choose the option that best supports a premium Pacific Northwest lodge experience.
