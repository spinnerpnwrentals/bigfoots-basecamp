# Agent Instructions

## Always Start Here

- Read PROJECT.md before making changes.
- Read any other available project documentation.
- Continue from the current local branch.
- If there are uncommitted changes, decide whether they are related to the requested task.
- If related, continue carefully.
- If unrelated, stop and explain.
- Never overwrite or discard user work without permission.

## Work Style

- Make focused, logical changes.
- Prefer one feature per commit.
- Preserve existing functionality.
- Keep the site static, fast, mobile-first, and accessible.
- Follow the Bigfoot's Basecamp brand and design principles.

## Scavenger Hunt Protection

Never break:

- localStorage progress
- clue rendering
- hints
- found/unfound buttons
- completion unlock
- confetti
- admin mode
- reset functionality

## Git Rules

- Commit completed work locally.
- Do not push to GitHub unless explicitly instructed.
- Do not publish to Netlify.
- Use clear commit messages.
- Keep commits small and reversible.

## Before Committing

- Validate affected pages locally.
- Check for obvious console/script errors.
- Confirm links still work.
- Summarize what changed.
